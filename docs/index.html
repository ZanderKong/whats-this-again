Chrome Manifest V3 Extension

这是啥来着

在网页里划选陌生概念，立即获得结合上下文的简洁解释；只保存真正有价值的回答，形成低干扰的本地阅读记忆。

安装使用 查看流程

Why it exists

让查词不打断阅读

阅读技术文档、论文、英文资料或行业文章时，最容易打断节奏的是陌生缩写、概念和上下文相关表达。这个插件把解释、追问和保存放在选中文本旁边，减少跳转、复制和整理成本。

Workflow

三步完成一次解释

01

划选概念

在普通网页中选中词语、缩写或短句，插件会在选区旁显示轻量入口。

02

解释或追问

点击入口自动解释；长按可自定义问题，回答后还能继续围绕同一上下文追问。

03

主动保存

临时回答默认不进入历史。确认有价值后再保存全文或节选，避免历史记录膨胀。

Scenarios

适合这些阅读场景

技术文档

快速解释 API、架构术语、库函数和配置项，保留与当前页面相关的解释。

论文与报告

结合前后文理解缩写、研究方法和关键概念，减少频繁切换搜索页面。

英文资料

翻译短语并说明语境含义，比单独查词更贴近当前文章。

长期学习

再次遇到保存过的词语时，以低干扰提示回看旧解释，形成可复用记忆。

Model support

连接你自己的模型服务

插件支持 DeepSeek、Kimi、火山方舟、智谱 GLM、MiniMax、硅基流动、OpenAI、OpenRouter、Groq、本地 Ollama、LM Studio，以及自定义 OpenAI-compatible 接口。

默认 prompt 会随语言自动切换，支持发送选区附近上下文，也可以关闭上下文，仅发送选中文本和问题。

Privacy

本地优先，明确保存

设置、API Key、保存的解释和批注历史存储在浏览器本地的 chrome.storage.local。批注原文、上下文和意见不会发送到项目作者的服务器。

插件只在拖放注入失败或用户主动复制时写入剪贴板，不读取剪贴板，也不自动发送消息。拖入第三方 AI 输入框后的内容由对应网站处理。

发起解释时，插件只会把选中文本、用户问题和可选上下文发送到用户配置的模型接口。

Install

通过开发者模式加载

  1. 打开 chrome://extensions 并启用 Developer mode。
  2. 点击 Load unpacked，选择包含 manifest.json 的项目根目录。
  3. 打开插件设置页，填写模型提供商、API Base URL、API Key 和模型名。
  4. 刷新目标网页，划选文本后开始解释。
