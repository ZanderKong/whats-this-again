Chrome Manifest V3 Extension

这是啥来着

Select unfamiliar concepts on a web page, get concise explanations grounded in nearby context, and save only the answers worth remembering.

Install View workflow

Why it exists

Look things up without breaking your reading flow

Technical docs, papers, English references, and industry articles often include acronyms, concepts, and expressions whose meaning depends on the surrounding text. This extension keeps explanations, follow-ups, and saving next to the selection so you spend less time copying, switching tabs, and organizing notes.

Workflow

Explain a selection in three steps

01

Select a concept

Select a word, acronym, or short phrase on a regular web page. A lightweight entry dot appears near the selection.

02

Explain or ask

Click to run the default explanation prompt, or long-press to ask a custom question. Continue with follow-ups in the same context.

03

Save intentionally

Temporary answers are not saved by default. Save the full answer or an excerpt only when it is useful.

Scenarios

Built for focused reading

Technical docs

Explain APIs, architecture terms, library functions, and configuration items while preserving page-specific context.

Papers and reports

Understand acronyms, methods, and key concepts from surrounding context without repeated search-tab switching.

English references

Translate phrases and explain what they mean in the current article, which is often more useful than a standalone dictionary lookup.

Long-term learning

When saved terms appear again, subtle reminders help you revisit previous explanations and build reusable reading memory.

Model support

Connect your own model endpoint

The extension supports DeepSeek, Kimi, Volcengine Ark, Zhipu GLM, MiniMax, SiliconFlow, OpenAI, OpenRouter, Groq, local Ollama, LM Studio, and custom OpenAI-compatible endpoints.

The default prompt follows the selected language automatically, page context can be included or disabled, and the interface supports Chinese and English with Auto, 中文, and English language modes.

Privacy

Local-first and explicit by default

Settings, API Keys, saved explanations, and annotation history are stored locally in chrome.storage.local. Annotation quotes, context, and comments are not sent to project-operated servers.

The extension writes to the clipboard only after failed drag insertion or an explicit copy action. It never reads the clipboard or sends messages automatically. Destination AI sites process content placed in their editors.

When you generate an explanation, the extension only sends selected text, your question, and optional page context to the model endpoint you configure.

The extension runs on ordinary http/https pages to provide selected-text explanations. chrome.storage.local is local browser storage, not a system password vault; do not select sensitive content, and use only trusted model services and browser environments.

Install

Load through Developer mode

  1. Open chrome://extensions and enable Developer mode.
  2. Click Load unpacked and choose the project root containing manifest.json.
  3. Open the extension settings page and enter provider, API Base URL, API Key, and model name.
  4. Refresh the target page, select text, and start explaining.
